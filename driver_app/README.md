# Driver App – Flutter Mobile Client

## Overview
This directory contains the Flutter-based mobile application for the embedded driver behavior monitoring system. The app provides a mobile interface for viewing live telemetry from the embedded device, reviewing historical driving data, and receiving alerts when aggressive driving behavior is detected.

The application is intended as a prototype client for demonstration of the end-to-end system and complements the embedded firmware, dashboard, and hardware subsystems in this repository.

---

## Core Functionality

### 1. Bluetooth Connectivity
The mobile app connects to the embedded device over Bluetooth Low Energy and receives live telemetry generated by the ESP32-based driver monitoring unit.

Typical streamed data includes:
- driver behavior classification
- confidence score
- speed
- RPM
- throttle
- longitudinal acceleration
- lateral acceleration
- yaw rate
- location data where available

---

### 2. Live Monitoring
The app provides a live view of driver behavior status and telemetry received from the device. This allows the user to observe current driving conditions in real time during system operation.

---

### 3. Historical Data Access
The app supports viewing previously recorded data originating from the backend/dashboard side of the system. This enables users to review past driving sessions and observe behavioral trends beyond the live stream.

---

### 4. Alerts and Feedback
The application provides immediate feedback when aggressive driving behavior is detected. Depending on the implemented flow, this may include:
- in-app notifications
- haptic feedback
- audible alerts / buzzer-style warning behavior on the mobile side

This supports timely awareness of unsafe driving events.

---

### 5. Authentication Flow
The current login flow is implemented as a prototype demonstration feature.

Authentication is mocked for demonstration purposes:
- credentials are hardcoded sample values
- no production authentication or secure credential handling is implemented in this prototype
- the included account names and passwords are examples only

This was sufficient for demonstrating app navigation, role-based flow, and system interaction during development.
